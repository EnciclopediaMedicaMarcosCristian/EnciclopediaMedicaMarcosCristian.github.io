Declaración de Accesibilidad

La “Enciclopedia Médica Marcos y Cristian” se ha comprometido a hacer accesible su sitio web de conformidad con el Real Decreto 1112/20184, de 7 de septiembre, sobre accesibilidad de los sitios web y aplicaciones para dispositivos móviles del sector público, transposición de la Directiva Europea 2016/21025 del Parlamento Europeo y del Consejo, de 26 de octubre de 2016, sobre la accesibilidad de los sitios web y aplicaciones para dispositivos móviles de los organismos del sector público

La presente declaración de accesibilidad se aplica al sitio web Enciclopedia Virtual de Medicina.

Situación de cumplimiento

Este sitio web es plenamente conforme con las Pautas de Accesibilidad para el Contenido Web (WCAG) 2.1, nivel AA.

  • Idioma principal: El idioma principal de todas las páginas está correctamente identificado, facilitando la navegación mediante lectores de pantalla.
  • Título de la página (WCAG 2.4.2): Cada página contiene un título descriptivo que refleja claramente su contenido.
  • Texto alternativo en imágenes (WCAG 1.1.1): Todas las imágenes relevantes incluyen texto alternativo que describe su contenido o función.
  • Navegación y enlaces (WCAG 2.4.4 y 3.2.4): Los enlaces permiten una navegación clara entre secciones y páginas. La imagen de la cabecera enlaza a la página principal.
  • Estructura y etiquetas semánticas (WCAG 1.3.1): La estructura de las páginas sigue una jerarquía de encabezados clara y utiliza etiquetas semánticas para mejorar la navegación.
  • Formularios (WCAG 3.3.1 y WCAG 3.3.2): Los formularios están diseñados con instrucciones claras, mecanismos de identificación de errores y etiquetas visibles para facilitar su uso.
  • Contraste (WCAG 1.4.3): El contraste entre texto y fondo cumple con los requisitos mínimos de 4.5:1 para garantizar la legibilidad.
  • Enfoque visible (WCAG 2.4.7): Los elementos interactivos tienen un estilo de foco visible para facilitar la navegación mediante teclado.
  • Orientación (WCAG 1.3.4): El diseño es adaptable a diferentes dispositivos, asegurando una visualización adecuada en móviles y pantallas de escritorio.
  • Uso de color (WCAG 1.4.1): No se depende únicamente del color como indicador de información; se incluyen íconos u otros marcadores visuales.
  • Tamaño del texto (WCAG 1.4.4): El texto es redimensionable hasta un 200% sin pérdida de contenido o funcionalidad.
  • Control del espaciado (WCAG 1.4.12): Se asegura suficiente espacio entre elementos textuales y campos interactivos para evitar superposición.
  • Navegación mediante teclado (WCAG 2.1.1): Todos los elementos interactivos son accesibles mediante teclado.
  • Enlaces "saltar a" (WCAG 2.4.1): Incluyen enlaces para saltar directamente al contenido principal de las páginas.
  • Media Queries (WCAG 1.4.10): El diseño responde adecuadamente a resoluciones de pantalla pequeñas, reorganizando el contenido para garantizar su legibilidad.
  • Fuente (WCAG 1.4.4): Las fuentes son escalables hasta un 200% sin pérdida de funcionalidad o contenido.

Contenido no accesible

El contenido que se recoge a continuación no es accesible por las siguientes razones, de acuerdo con las pautas WCAG:

a) Falta de conformidad con WCAG 2.1:

Falta de validación en formularios (WCAG 3.3.1): Algunos formularios no tienen identificadores claros de errores o instrucciones accesibles para los usuarios con discapacidades.

b) Carga desproporcionada:

Documentos PDF no accesibles: Algunos documentos PDF contienen texto no seleccionable, imágenes sin texto alternativo y un formato difícil de leer con lectores de pantalla.

c) Contenido fuera del ámbito de la legislación aplicable:

Contenido incrustado de terceros: Algunos elementos, como videos de plataformas externas (YouTube, Vimeo), no cumplen con los estándares de accesibilidad establecidos. Estos contenidos están fuera del ámbito de control del sitio web y pueden carecer de subtítulos, descripciones o controles de navegación accesibles.

Algunas de las alternativas accesibles son:

  • En el caso de los documentos PDF no accesibles, se deben crear sus versiones accesibles y eliminar las que no sean accesibles.
  • Para los videos sin subtítulos, se deben proporcionar transcripciones escritas que ofrecen la misma información.
